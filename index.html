Webci es una libreria para crear servidores webs y usar C para enviar y crear paginas html y tener mas control de las respuestas enviadas(o solicitudes) de un sitio web y enviar respuestas de acorde a ello.

Aún en desarrollo

Con webci se puede crear paginas html

#include <webci.h>
#include <sqlite3.h>
#include <curl/curl.h>

int main (){
    
    struct Server server;
    html html;
    
    server.port = 8080;
    server.buffer_file = 4096;
    open_server(&server);
    
    ini_html(&html);
    html.title("Page");
    html.html_o("lang=\"es\"");
    html.head_o(NULL);
    html.meta("charset=\"UTF-8\"");
    html.meta("http-equiv=\"X-UA-Compatible\" content=\"IE=Edge\"");
    html.head_c();
    html.body_o(NULL);
    html.h1(NULL,"Hola, mundo");
    html.body_c();
    html.html_c();
    
    html.send(&server,JS_ERRORS_OFF);
    
    return 0;
}

Lo que hace este codigo es usar la estructura html y la estructura Server para crear una pagina html y un servidor web. cada elemento html se lo llama desde la estructura html que realmente son funciones, algunas funciones o elementos html estan divididos en dos funciones siempre inician por el nombre del elemento seguido de un _o que es de open y la función de cierre es igual que lo anterior pero con la diferencia de que es _c de close para enviar la pagina html creada se usa la función html.send(estructura Server,macro para mostrar posibles errorres js);

Se pueden crear servidores webs

#include <webci.h>
#include <sqlite3.h>
#include <curl/curl.h>

int main (){
    
    struct Server server;
    
    server.port = 8080;
    server.buffer_file = 4024;
    open_server(&server);
    
    while(WEB_TRUE){
        send_response_server(&server,"index.html");
        printf("%s",get_response());
    }
    
    return 0;
}

En este codigo se utiliza la estructura Server la cual permite crear servidores webs de una forma mas sencilla, antes de abrir el servidor o de se hacen dos declaraciones previas las cuales son el .port y .buffer_file se crea un bucle principal que escuchara las solicitudes y enviara la pagina usando la funciòn send_response_server(estructura server, nombre del archivo html); y finalmente se usa la funciòn get_response() para ver la respuesta en la terminal recivida del navegador.
