Crear servidores

Con webci se facilita o se intenta de facilitar la creacion de los mismos, hay una struct llamada struct Server que es una estructura para crear los servidores webs cuyos datos son:

  • int server_fd
  • int new_socket
  • int valread
  • int client_socket
  • struct sockaddr_in address
  • struct sockaddr_in client_address
  • socklen_t client_address_length
  • int opt
  • int addrlen
  • int error
  • int port
  • int buffer_size

    • La estructura se declara así

    struct Server servidor;

    La estructura siempre sera global siempre cuando se quiere crear un servidor que trabaje con varias paginas html de lo contrario no es necesario. la forma de trabajar con esta estructura es simple en la función main() se escribe lo siguiente:

    servidor.port = 8090; servidor.buffer_size = 4024; start_server_file(&servidor);

    la función start_server_file() se encargara de inizializar los otros datos necesarios y solo recive como parametro una variable de la estructura Server. La función listen_server_file() es la encargada escuchar solicitudes de usuarios entrantes recive dos parametros: una variable de la struct Server y otra que es la pagina html a enviar siempre se crea un bucle while(WEB_TRUE) o while(1) donde vaya la función listen_server_file() cuyo segundo parametro sera la pagina inicio es decir: index.html
